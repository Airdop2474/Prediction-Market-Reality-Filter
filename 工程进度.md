# Prediction Market Reality Filter (PMRF) v0.3.0

## 系统架构

```
Polymarket API → fetch_markets()
       ↓
Scanner（并行，Semaphore=4）
  ├─ RSS 7源（并发 executor）
  ├─ GNews（executor）
  ├─ news_filter_service（相关性/来源/证据）
  └─ ai_analysis_service（LLM + deterministic fallback）
        ├─ market_semantics → YES/NO 条件解析
        ├─ clamp_probability（单次加权回归）
        ├─ anchor_probability（base_rate 历史锚定）
        ├─ passes_quality_gate
        └─ calculate_signal → STRONG_LONG/LONG/WATCHLIST/SHORT/STRONG_SHORT
              ↓
       analysis_audit.jsonl   ← JSONL 审计日志
       agent_memory.json      ← 含 signal + divergence
       market_cache.json      ← TTL 1h 缓存

反馈闭环（每天 22:00 UTC）：
  Polymarket 历史 API → auto_resolve → agent_memory + audit → Brier Score

交易日志（手动）：
  POST /trades/open → trade_journal.json
  POST /trades/close/{id} → 计算盈亏
  GET /trades/summary → 胜率 / 总盈亏 / 按信号类型
```

## 信号体系

| 信号 | 触发条件 |
|------|----------|
| `STRONG_LONG` | divergence > 20% AND confidence > 0.68 AND 过质量门 |
| `LONG` | divergence > 10% AND confidence > 0.50 AND 过质量门 |
| `WATCHLIST` | 质量门未通过，或偏差不足 |
| `SHORT` | divergence < -10% AND confidence > 0.50 AND 过质量门 |
| `STRONG_SHORT` | divergence < -20% AND confidence > 0.68 AND 过质量门 |

质量门（所有条件须同时满足）：
- confidence ≥ 0.50
- news_quality ≥ 0.40
- evidence_strength ≥ 0.20
- resolution_relevance ≥ 0.22
- conflict_score ≤ 0.65
- priced_in_risk ≤ 80

## 日常操作手册

```bash
# 启动服务
uvicorn app.main:app --reload

# 打开 Dashboard
http://localhost:8000/dashboard

# 每天流程
1. 早上：扫描市场 → 查看信号 → 决策入场 → 记录交易
2. 晚上：POST /resolve/auto → 查看 Brier Score 变化
3. 市场解决后：POST /trades/close/{id} → 记录盈亏

# 调试某个市场
GET /scan/debug?market_question=...&market_probability=50
```
